Introduction

Ad analytics is a free-of-charge offering that provides Brightcove customers with a transparent, meaningful, and comprehensive set of ad metrics. These metrics will focus on the following:

  • Troubleshooting: What ad configurations and partners are serving ads more efficiently? What are the technical errors preventing ad impressions?
  • Reporting/Impact: Tracking Ad Impressions, fill rate, and how the audience consumes the impressions. What are the impact of ad blockers and firewalls on your ad operation?

Availability

Ad Analytics are available free of charge to Brightcove customers to provide transparent, meaningful and comprehensive set of ad metrics. You can find them on the following platforms:

  • Video Cloud Studio
  • Beacon Studio

Ad metrics are available for the following:

  • Video On Demand (VOD) content
  • Ads served using Brightcove's Server-side Advertising (SSAI)

Navigation

  1. Open Video Cloud Studio.
  2. In the primary navigation, click Analytics.
  3. In the panel navigation, click Ad analytics.

Glossary of terms

Here is the glossary of terms for Ad analytics metrics:

Data availability

Data availability for these metrics is as follows:

  • The data must be retained and available for at least three years.
  • The data is available via queryable APIs in CSV or JSON format.
  • The data can be exported from the UI based on the filters and aggregations at an export time in CSV format.

Response time

All the data must be one hour fresh.

Metrics

Ad analytics should be able to collect and report on the following metrics:

  • Ads scheduled: Total number of scheduled ads across all the breaks.
  • Ads inserted: Number of creatives stitched into a VOD or Live Stream.
  • Ad impressions: Number of creatives that were played.
  • Ad completion: Number of creatives watched until completion.
  • Ad impression rate: Ad Impressions/Ad Slots.
  • Last ad view: The time of the most recent ad impression.

Dimensions

Ad Analytics must be able to collect, aggregate and filter on the following metrics:

  • Account: Account level metrics showing the total across all dimensions.
  • Video: Aggregation by video ID.
  • Ad config: SSAI Ad Configuration identifier. The UI should display the AdConfig Name and should be enriched with the adTag’s domain.
  • Ad provider: The domain of the ad server.

SSAI Performance

SSAI performance metrics help users track their ad information.

Metrics

SSAI performance metrics must be able to collect and report on the following metrics:

  • Ads request call count: Number of Ad request HTTP calls.
  • Ads tracking call count: Number of Ad Tracking HTTP calls.
  • Ad error call counts: Number of HTTP.
  • Ad error code: HTTP Error Response code.

Dimensions

SSAI performance metrics must be able to collect, aggregate and filter on the following metrics:

  • Ad config: SSAI Ad Configuration identifier. The UI displays the AdConfig Name and should be enriched with the adTag’s domain.
  • Ad provider: The domain of the Ad server.

Granularity

Here is the granularity for SSAI performances metrics:

  • Time aggregation: Hour, day, month.
  • Ads: 1K, 1M.
